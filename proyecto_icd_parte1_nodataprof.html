Introducción¶

El anlisis que a continuacin se presenta corresponde a una base de datos sobre procedimientos de Interrupcin Legal del Embarazo (ILE) realizados en las clnicas y hospitales de la Secretara de Salud de la Ciudad de Mxico, durante el periodo enero 2016 a octubre 2019. La base de datos est formado por 36 variables y 62,003 observaciones. Dicha informacin se encuentra disponible en la siguiente direccin: https://datos.cdmx.gob.mx/explore/dataset/interrupcion-legal-del-embarazo/table/

El objetivo que se establece es utilizar algoritmos de Machine Learning para desarrollar un modelo que sea capaz de predecir si la edad de la persona que llega a realizarse la interrupcin legal del embarazo pertenece a alguna de las siguientes catergoras:

  • Mayor o igual a 23 aos
  • Menor a 23 aos

En este contexto se plantea que el gobierno utilice esta informacin para disear campaas de prevencin de embarazos para las mujeres del primer grupo, es decir cuya edad es mayor o igual a 23 aos.

Tomando como base el data profiling anterior, se identifican de manera particular las siguientes cuestiones:

Variables Predictoras

  • autoref: el 72.7% de observaciones corresponden a "NaN".
  • desc_derechohab: la mitad de los datos estn catalogados como "ninguna" y el resto est dividido en 35 categoras.
  • religion: el 93 % de los datos corresponden a "religion catlica" o "ninguna".
  • alcaldia: existen 498 observaciones nicas.
  • menstrua: existen 1509 observaciones nicas.
  • consejeria: el 95% de las observaciones tiene el mismo dato: "si".
  • motiles : el 97.4% de las observaciones tiene el mismo dato: "interrupcion voluntaria".
  • h_ingreso: el 91% de las observaciones corresponde a "na" o "n/e".
  • desc_servicio: el 77% de las observaciones corresponde a "ile".
  • p_consent: las observaciones est concentradas en "nan" en un 71% y en "si" en un 28.7%.
  • sememb: indica el nmero de semanas de embarazo conforme a la ltima fecha de menstruacin. Esta variable incluye decimales, los cules podran considerarse como la fraccin de semana, es decir, 5.4 podra corresponder a 5 semanas de embarazo y 4 das. Si lo anterior fuera correcto, los decimales deberan variar entre 1 y 7; sin embargo se osberva que los decimales varan entre 1 y 9, en cuyo caso se puede pensar que se llev a cabo una ponderacin de forma que 5.4 corresponde a 5 semanas de embarazo y 3 das.
  • p_semgest: incluye nicamente valores enteros, es decir, indica el nmero de semanas de embarazo sin considerar los das.
  • p_diasgesta: es el complemento de la variable p_semgesta. Idealmente esta variable debera contener nmeros entre 0 y 6; pero las observaciones incluyen datos entre 7 y hasta 94 das.

A partir de las observaciones anteriores, se determin eliminar todas la variables enlistadas, excepto p_semgest. La variable fingreso es eliminada debido a que existen las variables ao y mes que generalizan la informacin contenida en la primera.En relacin a alc_o_municipio, el hecho de tener 500 observaciones nicas no muestra un patrn de comportamiento que agregue valor al anlisis. Lo concerniente a la variable fmenstrua reside en el nmero tan amplio de fechas incluidas, el cul podra resumirse al utilizar la variable p_semgest. La variable p_diasgesta representa un dato incompleto, ya que es el complemento de la variable p_semgest. Por otro lado, a pesar de que la variable sememb concentra ambos datos: semanas y das de embarazo, este dato se calcula a partir de la ltima fecha de menstruacin, por lo que se considera conveniente conservar p_semgest que es obtenida conforme al ultrasonido y descartar la primera. La eliminacin del resto de las variables se sigue a partir de las anomalas particulares indicadas arriba.

Imputacin de valores faltantes

3. Modelos¶

De manera general, en esta seccin se generarn tres modelos para llevar a cabo la prediccin planteada: Bosques Aleatorios, rboles de Decisin y XGBoost. Con el propsito de determinar los mejores parmetros para cada uno de ellos, se implementar un magic loop, dentro del cual ser incluida la validacin cruzada de 5 folds para evitar sobreajuste en el entrenamiento de los modelos. El desempeo de los modelos ser monitoreado a travs de la mtrica: recall.

Para este problema en concreto recall representa el cociente de mujeres cuya edad es mayor o igual a 23 en relacin al total de mujeres que tienen al menos 23 aos (mayor o igual a 23), es decir, recall permite identificar el porcentaje correcto de predicciones del grupo que tiene mayor presencia (mayor a 23 aos) en el conjunto de datos disponible.

# Definicin de la funcin seleccionar_modelos, que ser la encargada de seleccionar el modelo, dependiendo el nombre que reciba

def seleccionar_modelos(lista_modelos):
    
    config_modelos = np.array([])
    for modelo in lista_modelos:
        
        if modelo =='DECTREE':
            clasificador = DecisionTreeClassifier()
        if modelo =='RANDOMF':
            clasificador = RandomForestClassifier()
        if modelo =='XGBOOST':
            clasificador = GradientBoostingClassifier()
        
        config_modelos = np.append(config_modelos, clasificador)
        
    return config_modelos

# Definicin de la funcin magic_loop, la cual se encargar de correr los modelos con sus diferentes hyperparametros-
def magic_loop(configuracion_modelos, hyperParametros, X_train, Y_train, num_folds):
    
    resultados = np.array([])
    for i, clasificador in enumerate(configuracion_modelos):
        hyperParametros_modelo = hyperParametros[i]
        
        grid_search = GridSearchCV(clasificador
                                  ,hyperParametros_modelo
                                  ,scoring = 'recall'
                                  ,cv = num_folds
                                  ,n_jobs = -1
                                  ,verbose = 3
                                  )
        grid_search.fit(X_train, Y_train)
        cv_resultados = pd.DataFrame(grid_search.cv_results_)
        resultados_ordenados = cv_resultados.sort_values(by='rank_test_score', ascending=True)
        
        resultados = np.append(resultados, resultados_ordenados)
        
        # Se imprime los mejores hiperparmetros de todos los valores posibles que se incluyeron en el grid
        print('grid_search.best_params_: ', grid_search.best_params_)

        # Mejor score asociado a los modelos generados con los diferentes hiperparmetros,
        # corresponde al promedio de los scores generados durante la validacin cruzada
        print('grid_search.best_score_: ', grid_search.best_score_)

        
    return resultados

# Definicin de la funcin graficar_matrizConfusion que imprime y grafica la matriz de confusin.
# La normalzizacin puede aplicarse al especificar normalizar = True



def graficar_matrizConfusion(y_true, y_pred,
                          normalizar = False,
                          cmap = plt.cm.Blues):
    
    
    ######Clculo matriz de confusin
    matriz_confusion = confusion_matrix(y_true, y_pred)
    
    if normalizar:
        
        # Se calcula la matriz de confusin normalizada
        matriz_confusion = matriz_confusion.astype('float') / matriz_confusion.sum(axis=1)[:, np.newaxis]
        
        print("Matriz de Confusin Normalizada")
    else:
        print('Matriz de Confusin sin Normalizacin')

    print(matriz_confusion)
    
  
    ######Grfica de la matriz de confusin
    # Ttulo de la grfica de la matriz de confusin
    if normalizar:
        titulo = 'Matriz de Confusin Normalizada'
    else:
        titulo = 'Matriz de Confusin sin Normalizacin'

        
    # etiquetas de la variable target
    nombres_clases = unique_labels(y_pred)

    fig, ax = plt.subplots()
    im = ax.imshow(matriz_confusion, interpolation = 'nearest', cmap = cmap)
    ax.figure.colorbar(im, ax=ax)
    
    
    # Se quieren mostrar los valores de cada seccin de la matriz de confusin...
    ax.set(xticks = np.arange(matriz_confusion.shape[1]),
           yticks = np.arange(matriz_confusion.shape[0]),
           
           # ...y las etiquetas respectivas
           xticklabels = nombres_clases, yticklabels = nombres_clases,
           title = titulo,
           ylabel = 'True label',
           xlabel = 'Predicted label')

    
    # Se rotan las etiquetas y se fija su alineacin
    plt.setp(ax.get_xticklabels(), rotation=90, ha = "right", rotation_mode = "anchor")

    # Loop sobre las dimensiones de los datos y creacin de los textos de cada seccin de la matriz
    fmt = '.2f' if normalizar else 'd'
    thresh = matriz_confusion.max() / 2.
    for i in range(matriz_confusion.shape[0]):
        for j in range(matriz_confusion.shape[1]):
            ax.text(j, i, format(matriz_confusion[i, j], fmt),
                    ha = "center", va = "center",
                    color = "white" if matriz_confusion[i, j] > thresh else "black")
    fig.tight_layout()
    return ax
